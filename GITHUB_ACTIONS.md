# Настройка сборки через GitHub Actions (Android CI/CD)

Этот проект полностью подготовлен для сборки и автоматического тестирования в среде **GitHub Actions**.

---

## 🛠 Что было сделано:

1. **Исправлены ошибки компиляции:** Устранены все ошибки, связанные с вызовом функций `viewModel.setFilterLevel`, `viewModel.setBiometricsEnabled` и `viewModel.logOutYandex`. Проекту успешно возвращено полностью рабочее состояние (успешная сборка подтверждена локально).
2. **Сгенерирован Gradle Wrapper:** Созданы файлы `gradlew`, `gradlew.bat` и папка `gradle/wrapper/` с Gradle версии **9.3.1**, необходимые для выполнения сборок на облачных виртуальных машинах GitHub Runners.
3. **Создан сценарий сборки:** Добавлен файл конфигурации `.github/workflows/android.yml`, реализующий две независимые фазы (Jobs):
   * **build (Сборка Debug-версии):** Запускается на любой коммит или пул-реквест во все основные ветки (`main`, `master`, `develop`). Проверяет работоспособность кода, запускает юнит-тесты и выгружает артефакт в виде готового файла `app-debug.apk`.
   * **release (Сборка Release-версии):** Запускается **только при push-событии** в продакшн-ветку (`main` или `master`). Производит подпись релиза с помощью зашифрованных секретов (keystore), а также создает установочный `.apk` (Release) и полноценный бандл `.aab` для публикации в Google Play/RuStore.

## 🔐 Настройка секретов (GitHub Secrets)

Чтобы успешно собирать и подписывать релизную версию приложения для распространения, добавьте следующие секреты в настройках вашего GitHub репозитория (**Settings -> Secrets and variables -> Actions -> New repository secret**):

| Название секрета | Описание | Пример значения |
|---|---|---|
| **`SIGNING_KEYSTORE_BASE64`** | Весь файл вашего хранилища ключей (`.jks` / `.keystore`) закодированный в строку Base64. | *Инструкция по кодированию ниже* |
| **`STORE_PASSWORD`** | Пароль от файла хранилища ключей (Keystore Password). | `my_keystore_password_123` |
| **`KEY_PASSWORD`** | Пароль от конкретного ключа подписи (Key Password). | `my_key_password_123` |

### 🔑 Как закодировать Keystore в Base64 для секрета:

Откройте консоль и выполните команду в зависимости от операционной системы:

* **macOS / Linux:**
  ```bash
  base64 -i my-upload-key.jks | pbcopy
  ```
  *(Строка скопируется в буфер обмена, вставьте её напрямую в поле значения секрета `SIGNING_KEYSTORE_BASE64`)*

* **Windows (PowerShell):**
  ```powershell
  [Convert]::ToBase64String([IO.File]::ReadAllBytes("my-upload-key.jks")) | clip
  ```

## 🚀 Описание процесса сборки на GitHub Actions

1. При отправке изменений в ветки `develop`, `master` или `main` автоматически запустится рабочий процесс.
2. Система установит **Java JDK 17 (Zulu)**, закэширует зависимости Gradle для ускорения последующих запусков на ~70%, и подготовит конфигурационные `.env` файлы.
3. В случае успешного завершения сборки во вкладке **Actions** конкретного запуска вы сможете скачать готовые файлы:
   * **`debug-apk`** — для тестирования на устройствах и эмуляторах.
   * **`release-apk`** и **`release-bundle`** — финализированные и оптимизированные сборки для реального использования и сторов.
